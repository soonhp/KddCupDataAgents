# KDD Cup 2026 Data Agents 대회 심층분석 및 실행계획

## 1) 대회 핵심 목표 정리
- 대회는 **복합 데이터 분석 질문**에 대해, 에이전트가 스스로 계획을 세우고 다양한 데이터 소스(CSV/JSON/DB/문서)를 조합해 정답을 생성하는 능력을 평가한다.
- 공식 사이트 기준 대회 기간은 **2026-03-15 ~ 2026-08-09 (AoE, UTC-12)** 이다.
- 제출은 Docker 이미지 기반이며, 평가 환경에서 동일한 모델(Qwen3.5-35B-A3B)을 사용한다.

## 2) 기업형 멀티에이전트 조직 제안

### A. 조직 구조 (가상 전문가 팀)
1. **PM Agent (총괄 오케스트레이터)**
   - task 분해, 우선순위 결정, SLA/예산/리소스 관리.
2. **Planner Agent (분석 설계자)**
   - DAG형 실행계획 생성(분기/병합/루프 포함).
3. **Data Profiling Agent (데이터 진단)**
   - context 파일 구조 파악, 품질 점검, 결측/키/단위 확인.
4. **SQL Analyst Agent (정형 데이터 전문가)**
   - SQLite 스키마 탐색, 질의 최적화, 조인/집계.
5. **Python Analyst Agent (고급 계산 전문가)**
   - 통계/시계열/커스텀 로직 실행 및 검증.
6. **Doc Reasoning Agent (비정형 문서 해석)**
   - 정책/정의/조건 문서 추출 및 규칙화.
7. **Verifier Agent (검증·반증 담당)**
   - 후보 답안에 반례 탐색, 단위/범위/정합성 점검.
8. **Answer Contract Agent (출력 계약 준수)**
   - 최종 CSV 컬럼/타입/정규화 규칙 준수 확인.

### B. 실행 원칙
- **Plan-Execute-Verify-Repair** 루프를 기본으로 사용.
- 정답 생성 전 최소 1회 이상 **독립 검증 경로**를 실행.
- 난이도/데이터 유형별 라우팅 정책 사용(예: DB 중심 문제는 SQL Agent 우선).

## 3) 목표(상위권) 달성을 위한 단계별 운영계획

### Phase 0. 기반 구축
- 규칙 준수 체크리스트 자동화(환경변수, 파일 I/O, 출력 형식, timeout).
- 실험 추적(프롬프트/툴 호출/정답/실패원인) 표준화.

### Phase 1. 강한 베이스라인
- Starter Kit ReAct를 유지하되, 다음을 즉시 추가:
  - 동적 파일 인덱싱
  - SQL/Python 도구 선택 라우터
  - 중간결과 캐시
  - 실패 시 재시도 정책

### Phase 2. 멀티에이전트화
- Planner + Specialist + Verifier 3계층 구조 적용.
- 병렬 분기 실행(가능한 하위 질문 동시 처리)으로 시간 단축.

### Phase 3. 점수 최적화
- 답안 정규화(숫자 반올림, 날짜 포맷, 단위 변환) 모듈 강화.
- 공용 에러 택소노미 기반 리그레션 테스트 운영.

## 4) KPI 제안
- Public score
- Task 성공률(완전 정답률)
- 도구 호출 성공률(SQL/Python)
- 평균 스텝 수/평균 지연시간
- 포맷 오류율(prediction.csv 계약 위반)

## 5) 리스크와 대응
- 리스크: 모델 고정(평가 시 동일 모델)으로 모델빨 제한.
- 대응: 모델 의존 대신 **도구 사용 정확도 + 검증 루프** 중심으로 성능 확보.
- 리스크: 이기종 데이터 결합 오류.
- 대응: 중간 산출물 스키마 표준(JSON schema) 강제.
