# [노션 공유본] KDD Data Agents 2026 전략 문서

## TL;DR
- 목표: 상위권 달성을 위한 기업형 멀티에이전트 체계 구축
- 핵심: 문제분해(DAG) + 도구선택(SQL/Python/문서) + 검증루프 + 출력정규화
- 즉시 실행: Starter Kit 기반에서 Router/Verifier/Normalizer 추가

## 1. 대회 분석
- 이 대회는 이기종 데이터(CSV/JSON/DB/문서) 조합 문제를 에이전트가 자율적으로 해결하는 능력을 평가.
- 공식 평가 환경은 모델/런타임 제약이 있으므로 시스템 설계력이 중요.

## 2. 팀(에이전트) 구성
- PM / Planner / Data Profiling / SQL / Python / Doc / Verifier / Answer Contract

## 3. 실행 로드맵
- Phase0: 규칙 준수 자동검증
- Phase1: 강한 baseline
- Phase2: 멀티에이전트 확장
- Phase3: 점수 최적화(정규화 + 회귀테스트)

## 4. Starter Kit 인사이트
- 실행 환경은 정상이나, 기본 상태에서 dataset 경로가 비어 있어 별도 데이터 주입 필요.
- 상위권을 위해 단일 ReAct를 멀티에이전트 오케스트레이션으로 확장해야 함.

## 5. 데이터셋 전략
- 동적 context 탐색 필수
- SQL/Python 교차검증
- 포맷 정합성 자동검사로 감점 방지

## 6. 이번 주 액션 아이템
1) AGENTS 규칙 템플릿 저장소 반영
2) Router + Verifier 최소기능 구현
3) 20개 task 샘플 실험으로 에러 택소노미 작성
4) Slack 데일리 리포트 자동 포맷 운영
