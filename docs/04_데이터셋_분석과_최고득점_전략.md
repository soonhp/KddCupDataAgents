# 데이터셋 분석 및 최고득점 전략

## 1) 데이터셋 특성(규칙/문서 기반)
- task별로 `task.json + context/` 구조를 가진다.
- `context/`는 CSV/JSON/SQLite/문서 등 조합이 매 task마다 달라질 수 있다.
- 즉, 고정 파이프라인보다 **동적 탐색 + 적응형 계획**이 필수다.

## 2) 실패 유형 가설
1. 파일 탐색 누락(존재하는 데이터 소스를 못 읽음)
2. DB 조인 키 오해/단위 해석 오류
3. 문서 규칙 반영 누락(예외 조건 미적용)
4. 최종 답안 포맷 불일치(컬럼/타입/정규화)

## 3) 최고득점 아키텍처 제안

### A. 4계층 파이프라인
1. **Discover**: 파일/스키마/문서 인덱싱
2. **Plan**: 서브질문 DAG 생성
3. **Execute**: SQL/Python/문서추출 병렬 수행
4. **Verify**: 반증 테스트 후 정답 확정

### B. 정답 신뢰도 스코어링
- 각 후보 답안에 대해 근거 수, 검증 통과율, 단위 일관성 점수 계산.
- 최고 점수 후보를 최종 제출.

### C. 실행 시간 최적화
- task difficulty 기반 max_steps 가변화
- heavy task만 병렬 worker 확대
- 중간결과 캐시 재사용

## 4) 실험 설계
- 축 1: 프롬프트 전략 (직접질의 vs 계획유도)
- 축 2: 툴 정책 (SQL 우선 vs Python 우선)
- 축 3: 검증 강도 (단일/이중/삼중)
- 축 4: 정규화 정책 (느슨/엄격)

평가 지표:
- Task 정답률
- 컬럼 매칭 성공률
- 값 정규화 성공률
- 평균 latency/timeout 비율

## 5) 운영 자동화
- 매 실행마다 `trace.json`, `prediction.csv`, 요약 리포트 자동 저장.
- 실패 task는 원인코드(탐색/계획/실행/검증/포맷) 태깅.
- 주간 회고에서 에러 상위 20% 유형 우선 개선.
